Project: cancer_detection_project

Objective:
The objective of this project is to build a machine learning model
(using a Convolutional Neural Network) that can analyze medical
images and classify them as either benign (non-cancerous) or
malignant (cancerous). The project demonstrates how image data
can be processed, used to train a model, and applied to make
predictions on new, unseen images.

Folder and File Descriptions:

data/
    Contains the dataset used for training and validation.

    train/
        Contains labeled training images.
        benign/      -> Images of non-cancerous cells.
        malignant/   -> Images of cancerous cells.
        normal/      -> (Optional/unused) Extra class; should be
                        removed unless doing 3-class classification.

    validation/
        Contains labeled validation images used to evaluate the
        model during training.
        benign/      -> Validation images (non-cancerous).
        malignant/   -> Validation images (cancerous).
        normal/      -> (Optional/unused) Same note as above.

outputs/
    Stores results generated by the model, such as plots,
    trained model files, or logs.

    cancer_model.h5 is a trained neural network saved to the computer's
    disk.

    Think of cancer_model.h5 in the following way:
    Before training -> your model is just an empty brain
    After training -> it has learned patterns from images

    .h5 is the brain saved as a file.

    .h5 stands for HDF5 (Hierarchical Data Format version 5).

    This is a file format that stores:
    • Model architecture.
    • Learned weights.
    • Training configuration.

    cancer_model.h5 shouldn't be opened manually.
    It must be loaded within a program with source code.

src/
    Contains the main source code notebooks.

    data_loader.ipynb
        Loads and prepares image data.
        Verifies folder structure and creates data generators.

    model.ipynb
        Defines and builds the CNN model architecture.

    predict.ipynb
        Loads a trained model and makes predictions on new images.

    train.ipynb
        Runs the full training pipeline, including fitting the model
        and evaluating performance.

test_images/
    Contains sample images used to test the trained model.

    test_image.jpg
        Example image used for prediction after training.

detecting_mutated_Cells.ipynb
    A combined or earlier notebook that may include the full
    pipeline (data loading, training, and prediction) in one place.

requirements.txt
    Lists all Python dependencies required to run the project.

README.md
    Provides an overview of the project, setup instructions,
    and usage details for users.

How to generate requirements.txt:

Run the following command in your terminal:

pip freeze > requirements.txt

This will automatically list all installed Python packages and
their versions into the requirements.txt file.
